# ArtStore 网站使用手册

这是一个艺术藏品展示网站。访客可以浏览已上架的藏品、图片、价格、简介和库存状态。

这份手册给**网站负责人和日常运营人员**使用：你不需要懂代码，不需要登录服务器，也不需要安装软件。照着页面点击即可维护藏品。

> 网站由企业服务器提供服务，藏品资料和图片保存在 Supabase。
> **日常新增、修改或下架藏品后，不需要重新发布网站。**

---

## 先判断你要做什么

| 你想做的事 | 去哪里操作 | 看哪一节 |
| --- | --- | --- |
| 新增一件藏品、改价格、换图片、上架或下架 | Supabase Dashboard | [日常维护藏品](#日常维护藏品) |
| 修改网站的文字、排版、功能或样式 | 请技术人员或 AI 修改代码 | [发布网站新版本](#发布网站新版本) |
| 第一次让网站上线 | 请负责人、服务器管理员和技术人员一起完成 | [第一次上线](#第一次上线) |
| 网站无法打开或显示异常 | 先记录现象和网址，不要自行删除内容 | [网站出问题怎么办](#网站出问题怎么办) |

## 负责人需要保管的东西

请用企业密码管理器保存，并至少让两位负责人拥有受控访问权限：

- GitHub 仓库管理员账号；
- Supabase 项目管理员账号；
- 域名服务商账号；
- 云服务器控制台账号；
- 正式网站域名；
- 当前网站的发布版本号和上线记录。

### 绝对不要分享的内容

不要把以下内容发给陌生人、粘贴到聊天窗口、提交到 GitHub 或交给 AI：

```text
服务器密码
SSH 私钥文件
Supabase 数据库密码
Supabase service_role / secret key
GitHub token
域名服务商密码或 API token
支付平台 secret key
完整 .env.production 文件
```

如果任何一项意外泄露，立即由对应平台的管理员撤销或更换。

## 第一次上线

首次上线需要有服务器权限的人操作。运营人员不需要独自完成，也不需要理解服务器命令。

### 上线前准备

请负责人先确认：

- [ ] 已购买域名，例如 `gallery.example.com`；
- [ ] 已准备一台 Ubuntu 24.04 LTS 云服务器，并有固定公网 IP；
- [ ] 已有 GitHub 仓库管理员权限；
- [ ] 已创建 Supabase 项目；
- [ ] 已准备至少一位服务器管理员；
- [ ] 已准备至少一位日常运营人员；
- [ ] 上述账号已开启 MFA（多因素认证）。

### 交给技术人员或 AI 的操作

将下面这段文字复制给技术人员或 AI，并提供本项目文件夹：

```text
请为 ArtStore 完成首次正式服务器部署。

我已准备好：Ubuntu 24.04 服务器、域名、GitHub 仓库和 Supabase 项目。
请严格阅读 docs/ai-technical-reference.md，并按其中“首次部署 runbook”执行。

要求：
- 先提出执行计划，让我确认后再登录服务器、改 DNS、改防火墙、创建密钥或启动生产容器。
- 不要读取、打印、提交 .env.production、SSH 私钥、数据库密码或任何 token。
- 不要执行 git reset --hard、git clean、rm -rf、docker compose down -v 或 docker volume rm。
- 完成后给我一份验收结果：网站网址、HTTPS 状态、当前发布版本、首页和详情页刷新测试结果。
```

### 上线完成后，你应该看到什么

技术人员或 AI 完成后，请在电脑和手机浏览器打开：

```text
https://你的正式域名
```

确认：

- [ ] 浏览器地址栏显示 HTTPS 锁标志；
- [ ] 首页能打开；
- [ ] 已上架藏品显示正确；
- [ ] 点击一件藏品可以打开详情页；
- [ ] 在详情页按刷新，页面不会显示 404；
- [ ] `All works` 和 `Available` 可以筛选；
- [ ] `Menu` 可以打开；
- [ ] `Continue viewing` 可以跳到下一件；
- [ ] 手机浏览器也能正常显示；
- [ ] 页脚联系方式正确。

把上线日期、网站域名和当前版本号记录在企业内部文档中。

## 日常维护藏品

这部分是运营人员最常用的内容。所有操作在 **Supabase Dashboard** 完成。

> **不要在 Supabase SQL Editor 重新执行仓库中的 SQL 文件。** 日常新增、修改或下架藏品只使用 Storage 和 Table Editor。SQL 仅由技术人员依照技术参考操作；其中 `supabase/000_reset_gallery_inventory.sql` 会永久删除所有藏品数据，不能用于正式日常维护。

### 上传图片前的准备

请尽量使用：

- JPEG、WebP 或 AVIF 格式；
- 图片宽度不超过 2000px；
- 单张图片尽量小于 1 MB；
- 使用藏品编号命名，例如 `2026050301.jpg`。

网站会按照上传图片原本的方向和比例展示，不会裁切图片。

### 新增一件藏品

1. 登录 Supabase Dashboard；
2. 点击左侧 **Storage**；
3. 打开 `artwork-images`；
4. 点击 **Upload file** 上传图片；
5. 上传完成后，打开该图片右侧菜单；
6. 选择 **Get public URL** 或 **Copy URL**；
7. 复制图片网址；
8. 点击左侧 **Table Editor**；
9. 打开 `artworks` 表；
10. 点击 **Insert → Insert row**；
11. 填写资料；
12. 在 `image_url` 粘贴第 7 步复制的网址；
13. 勾选 `is_published`；
14. 点击保存；
15. 刷新正式网站，确认作品显示正常。

### 每个字段怎么填

| 字段 | 填什么 | 示例 |
| --- | --- | --- |
| `inventory_number` | 唯一藏品编号，不能与旧编号重复 | `2026050301` |
| `title` | 英文名称 | `Puppet (Yoke Thay)` |
| `title_zh` | 中文名称，可留空 | `提线木偶` |
| `origin` | 产地或来源，可留空 | `Myanmar (Burma) · 缅甸` |
| `material` | 材质 | `Wood, gold thread, fabric` |
| `technique` | 工艺说明，可留空 | `Handcrafted using multi-technique` |
| `dimensions` | 尺寸 | `72 × 46 × 19 cm` |
| `price` | 对外显示价格 | `￥15,000` |
| `inventory_status` | 从下拉列表选择状态 | `available` |
| `is_published` | 是否在网站展示 | 勾选表示公开展示 |
| `image_url` | 刚复制的图片网址 | 粘贴 URL |
| `sort_order` | 首页排序数字，数字小的靠前 | `1` |

### 上架、下架和售出

| 你要做的事 | 在 `artworks` 表中怎么改 | 网站会怎样 |
| --- | --- | --- |
| 上架 | 勾选 `is_published` | 访客可以看到。 |
| 下架 | 取消勾选 `is_published` | 访客看不到。 |
| 暂时保留 | `inventory_status` 选 `reserved`，保持勾选上架 | 访客可见，详情显示 On hold。 |
| 已售但保留展示 | `inventory_status` 选 `sold`，保持勾选上架 | 访客可见，但不会出现在 Available 筛选中。 |
| 已售且不再展示 | 选 `sold`，并取消勾选上架 | 访客看不到。 |

### 更换图片

1. 在 Storage 上传新图片；
2. 复制新图片的 Public URL；
3. 在 `artworks` 表找到对应藏品；
4. 用新网址替换 `image_url`；
5. 保存；
6. 强制刷新浏览器：

```text
Mac：Command + Shift + R
Windows：Ctrl + Shift + R
```

确认新图显示正确后，再考虑删除旧图片。删除前先确认没有其他藏品使用该图片。

### 什么时候需要重新发布网站？

**通常不需要。** 下列操作保存后刷新网站即可：

- 新增或编辑藏品；
- 改价格；
- 改库存状态；
- 上架或下架；
- 上传或更换图片；
- 调整 `sort_order` 排序。

只有修改网站页面、交互、字体、版式、代码或数据库结构时，才需要走下一节的发布流程。

## 发布网站新版本

只有网站负责人批准后，技术人员或 AI 才能发布新版本。

### 你需要做什么

1. 写清楚需求，例如“把首页介绍文字改成……”“新增一个联系方式”“调整详情页价格位置”；
2. 要求技术人员或 AI 先提出修改计划；
3. 审核页面效果和改动说明；
4. 确认 GitHub 的 **Build** 检查通过；
5. 明确说“批准发布到生产服务器”；
6. 发布后按下方检查单验收。

### 给 AI 的修改任务模板

```text
这是 ArtStore 艺术藏品网站。

我的需求：<在这里写需求>

请先阅读 README.md、docs/ai-technical-reference.md，以及与需求相关的源码。
先给出最小修改方案，等我确认后再改代码。

限制：
- 不要读取、打印、修改或提交 .env.local、.env.production、SSH 私钥或任何密码。
- 不要登录服务器、修改 DNS、防火墙、Supabase 权限或生产数据，除非我明确批准。
- 不要创建 commit、推送 GitHub 或发布生产网站，除非我明确批准。
- 不要执行 git reset --hard、git clean、rm -rf、git push --force、docker compose down -v 或 docker volume rm。
- 修改后运行 npm run build，并向我说明测试结果。
```

### 发布后检查

技术人员或 AI 报告发布完成后，请在浏览器检查：

- [ ] 打开正式网站没有报错；
- [ ] 本次修改的内容正确；
- [ ] 首页和详情页都能打开；
- [ ] 在详情页刷新不会出现 404；
- [ ] 手机浏览器显示正常；
- [ ] 已上架藏品仍能显示；
- [ ] 当前版本号和发布日期已记录。

## 网站出问题怎么办

### 网站打不开、出现证书警告或页面全部空白

1. 截图保存错误信息；
2. 复制浏览器地址；
3. 记录发现问题的时间；
4. 不要删除服务器文件、Docker 容器、图片或 Supabase 数据；
5. 联系技术人员或 AI，并发送：网址、截图、时间和“最近是否发布过新版本”。

### 只有一件藏品不显示或图片打不开

1. 在 Supabase `artworks` 表确认该记录的 `is_published` 已勾选；
2. 确认 `image_url` 粘贴的是 Storage 复制的 Public URL；
3. 在浏览器新标签打开该图片网址；
4. 强制刷新网页；
5. 仍不正常时，把藏品编号和截图发给技术人员。

### 发布新版本后发现问题

不要在服务器上自行编辑或删除文件。请技术人员回滚到上一个已验证版本。

你需要提供：

- 发生问题的时间；
- 正式网站网址；
- 问题截图和操作步骤；
- 最近一次发布的大致时间；
- 是否影响所有访客，还是只影响某个页面。

技术人员必须遵照 [`docs/ai-technical-reference.md`](docs/ai-technical-reference.md) 的“发布与回滚 runbook”完成回滚和验证。
